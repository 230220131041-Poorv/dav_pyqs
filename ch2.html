Definition

A cluster is a group of data points that are more similar to each other than to data points in other groups. Cluster analysis is an unsupervised learning technique that groups unlabeled data into natural clusters without any prior knowledge of group labels.

Algorithms for Identifying Clusters

K-Means

Partitions data into K clusters by minimizing intra-cluster variance. Fast and simple.

Hierarchical Clustering

Builds a dendrogram by merging (agglomerative) or splitting (divisive) clusters progressively.

DBSCAN

Density-based. Groups dense regions; marks sparse points as outliers/noise. No K needed.

KDE (Kernel Density Estimation)

Estimates data density; peaks of density curve correspond to cluster centers.

Gaussian Mixture Models

Probabilistic model assuming data is from a mixture of Gaussian distributions.

Mean Shift

Shifts data points towards the mode of density. Does not require K to be specified.

Applications of Cluster Analysis

  • Customer Segmentation: Group customers by purchasing behaviour for targeted marketing
  • Document Clustering: Group similar news articles or research papers
  • Image Segmentation: Partition image pixels into meaningful regions
  • Anomaly Detection: Isolated points not in any cluster are outliers
  • Bioinformatics: Group genes with similar expression patterns

Win'23

Characteristics of Good Clustering

  1. High Intra-cluster Similarity: Points within the same cluster must be very similar (small intra-cluster distance) — tight, compact clusters.
  2. Low Inter-cluster Similarity: Points in different clusters must be very dissimilar (large inter-cluster distance) — clusters are well-separated.
  3. Scalability: Should work efficiently on very large datasets without degrading in performance.
  4. Ability to Handle Different Data Types: Should handle numerical, categorical, and mixed-type data.
  5. Robustness to Outliers/Noise: Should produce meaningful clusters even when the data contains noisy or extreme points.
  6. Insensitivity to Input Order: Results should not depend on the order in which data points are processed.
  7. Minimal Parameters: Should work without requiring users to specify many configuration values (ideally not even K).
  8. Interpretability: Clusters produced should be meaningful and explainable to domain experts.

Q 4Discuss Estimating Clusters with Kernel Density Estimation (KDE).4 M
Sum'23

What is KDE?

Kernel Density Estimation (KDE) is a non-parametric technique to estimate the probability density function (PDF) of a dataset. Instead of assuming any fixed distribution, KDE places a smooth kernel function at each data point and sums them to build a density curve. In clustering, the local peaks (maxima) of this density curve identify cluster centers.

KDE Formula

f̂(x) = (1/nh) × Σ K((x − xᵢ)/h) Where: n = number of data points h = bandwidth (smoothing parameter) K = kernel function (usually Gaussian) xᵢ = individual data points

How KDE Identifies Clusters — Steps

  1. Place a smooth Gaussian kernel centered at each data point
  2. Sum all kernels across the entire data space to get the density estimate f̂(x)
  3. Plot the density curve — high peaks = dense regions = cluster centers
  4. Identify all local maxima (peaks) in the density function
  5. Assign each data point to the nearest peak — these are the clusters

Role of Bandwidth (h)

Small h (undersmoothing)

Many sharp narrow peaks → too many clusters detected. Very sensitive to noise.

Large h (oversmoothing)

Few broad peaks → fewer clusters found. May merge truly distinct groups.

KDE — Individual Kernels Summed to Form Density Curve with 3 Peaks (Clusters)
📝 Key idea: Peaks in density curve = cluster centers. Bandwidth h is the key parameter controlling the number of clusters found.

What is DBSCAN?

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) is a density-based clustering algorithm. It groups together points that are closely packed (high density) and marks points in sparse, low-density regions as noise/outliers. It does not require specifying the number of clusters in advance.

Two Key Parameters

ε (epsilon)

The radius of the neighbourhood around each point. Two points are neighbours if their distance ≤ ε.

MinPts

Minimum number of points required within the ε-neighbourhood to form a dense region (core point).

Three Types of Points

  • Core Point: Has at least MinPts neighbours within distance ε. Forms the interior of a cluster.
  • Border Point: Has fewer than MinPts neighbours within ε, but lies within ε of a core point. Lies on the edge of a cluster.
  • Noise Point (Outlier): Neither core nor border point. Does not belong to any cluster.
DBSCAN — Core, Border, and Noise Points with ε Neighbourhood

DBSCAN Algorithm — Steps

  1. For each unvisited point P, mark it as visited
  2. Find all points within distance ε (the neighbourhood)
  3. If neighbourhood size ≥ MinPts → P is a Core Point; start a new cluster
  4. Expand the cluster by recursively adding all density-reachable points
  5. If size < MinPts → mark P as Border or Noise (temporary)
  6. Continue until all points are visited

Advantages vs Disadvantages

AdvantagesDisadvantages
No need to specify number of clusters KSensitive to choice of ε and MinPts
Finds arbitrarily shaped clustersStruggles with varying density clusters
Automatically identifies outliers/noisePoor performance on high-dimensional data

Q 6Discuss types of clustering.7 M
Win'23

Overview

Clustering algorithms can be broadly classified into the following categories based on their approach to grouping data:

Types of Clustering — Taxonomy

1. Partitioning Clustering

Divides data into exactly K non-overlapping subsets. Each data point belongs to exactly one cluster. Requires K to be specified in advance. Optimizes an objective function (minimize WCSS).

  • Examples: K-Means, K-Medoids (PAM)

2. Hierarchical Clustering

Builds a tree-like structure (dendrogram) showing how clusters merge (agglomerative, bottom-up) or split (divisive, top-down) at different levels of similarity. No K needed.

  • Examples: Ward's linkage, Single/Complete/Average linkage

3. Density-Based Clustering

Groups data points in regions of high density. Can find arbitrarily shaped clusters and naturally identifies noise/outlier points. Does not require K.

  • Examples: DBSCAN, OPTICS, DENCLUE

4. Grid-Based Clustering

Divides the data space into a finite grid of cells. Performs clustering on the grid rather than individual points — making it very fast and scalable for spatial data.

  • Examples: STING, CLIQUE, WaveCluster

5. Model-Based Clustering

Assumes data is generated from a mixture of statistical distributions. Uses the Expectation-Maximization (EM) algorithm to estimate parameters.

  • Example: Gaussian Mixture Models (GMM)

6. Fuzzy Clustering

Each data point has a degree of membership (probability) to each cluster rather than a hard binary assignment. A point can partially belong to multiple clusters.

  • Example: Fuzzy C-Means (FCM) — a point could be 70% in Cluster 1 and 30% in Cluster 2

Q 7Describe K-Means Clustering Algorithm with example.7 M

What is K-Means?

K-Means is a partitioning clustering algorithm that divides a dataset into exactly K clusters by iteratively assigning each data point to the nearest centroid and recalculating centroids until the assignments stabilize (converge).

Objective: Minimize Within-Cluster Sum of Squares (WCSS):

WCSS = Σₖ Σᵢ∈Cₖ ||xᵢ − μₖ||² Where μₖ = centroid (mean) of cluster k

Algorithm Steps

  1. Initialize: Randomly select K data points as initial cluster centroids (μ₁, μ₂, ..., μₖ)
  2. Assign: Assign every data point to the nearest centroid using Euclidean distance
  3. Update: Recalculate each centroid as the mean of all points now assigned to it
  4. Convergence check: If centroids have not changed (or change < threshold), stop
  5. Repeat: If centroids changed, return to Step 2
K-Means — 4 Stages of the Algorithm (K = 2)

Choosing K — Elbow Method

Plot WCSS vs number of clusters K. The "elbow" point — where the rate of decrease sharply slows — is the optimal K value.

Advantages & Limitations

AdvantagesLimitations
Simple, fast, and scalableK must be specified in advance
Works well for spherical clustersSensitive to initial centroid placement
Easy to implement and interpretCannot find non-spherical clusters
Scales to large datasetsAffected by outliers

Q 8Describe distance measures used in data science.4 M
Win'23

Why Distance Measures Matter

Distance measures quantify the dissimilarity between two data points. They are fundamental to clustering (K-Means, hierarchical, k-NN) — the choice of distance metric directly affects the quality of results.

1. Euclidean Distance

Most common. Straight-line ("as the crow flies") distance between two points in n-dimensional space.

d(A,B) = √[Σ(Aᵢ − Bᵢ)²] Example: d((1,2),(4,6)) = √[(4−1)² + (6−2)²] = √(9+16) = 5

2. Manhattan Distance (City Block / L1)

Sum of absolute differences along each axis. Like navigating a city in a grid — only horizontal and vertical moves.

d(A,B) = Σ|Aᵢ − Bᵢ| Example: d((1,2),(4,6)) = |4−1| + |6−2| = 3 + 4 = 7

3. Minkowski Distance

Generalized form. Euclidean (p=2) and Manhattan (p=1) are special cases.

d(A,B) = (Σ|Aᵢ − Bᵢ|ᵖ)^(1/p)

4. Cosine Similarity

Measures the angle between two vectors (not their magnitude). Used in text mining and document clustering.

cos(θ) = (A·B) / (||A|| × ||B||)

5. Hamming Distance

Count of positions where two equal-length strings differ. Used for categorical data and binary features.

What is Random Forest?

Random Forest is an ensemble supervised learning algorithm that builds many decision trees and combines their outputs (majority vote for classification, average for regression) to produce a more accurate and robust prediction. It is based on Bagging (Bootstrap Aggregating).

Three Key Ideas

1. Bootstrap Sampling

Each tree trains on a different random sample of the training data drawn with replacement.

2. Random Feature Selection

At each split node, only a random subset of features (m features where m << total features) is considered. This decorrelates the trees.

3. Majority Voting (Ensemble)

For classification: the class predicted by the most trees wins. For regression: average of all tree outputs.

4. Out-of-Bag (OOB) Estimate

Data not included in a tree's bootstrap sample is used to estimate that tree's error — a free internal validation mechanism.

Working — Step by Step

  1. From N training samples, create B bootstrap samples (random subsets with replacement)
  2. For each bootstrap sample, grow a decision tree. At each node, randomly select m features out of all F features and choose the best split from only those m
  3. Grow each tree fully to maximum depth — no pruning at this stage
  4. To predict for a new data point, pass it through all B trees
  5. Aggregate: majority vote (classification) or average (regression) gives the final answer
Random Forest — Ensemble of B Decision Trees with Majority Vote

Applications

  • Medical diagnosis — classifying diseases from patient data
  • Credit scoring and fraud detection in banking
  • Stock price movement prediction
  • Feature importance ranking (built-in feature selection)
  • Remote sensing and satellite image classification

Advantages & Disadvantages

AdvantagesDisadvantages
High accuracy — reduces overfitting compared to single treeSlower prediction (many trees to evaluate)
Handles missing values naturallyNot easily interpretable ("black box")
Works for both classification and regressionRequires more memory and computation
Provides built-in feature importance scoresMay overfit on very noisy datasets
📝 Three keywords to remember: Bootstrap sampling + Random feature selection + Majority voting. This is one of the most frequently tested 7-mark topics.

What is a Decision Tree?

A Decision Tree is a supervised learning algorithm that uses a tree structure of feature tests to classify data. Each internal node tests a feature, each branch is an outcome, and each leaf node gives the final class label.

Key Concepts

  • Root Node: Top node — represents the most important feature (best split)
  • Internal Node: Tests a feature condition (e.g., Age > 30?)
  • Branch: Outcome of the feature test (Yes/No)
  • Leaf Node: Final class label / decision

Split Criteria

Information Gain (ID3 algorithm)

Choose feature that maximally reduces entropy (uncertainty).

IG = Entropy(parent) − Weighted Entropy(children)

Gini Index (CART algorithm)

Measure impurity of a node. Lower Gini = purer = better split.

Gini = 1 − Σ(pᵢ)²

What is Tree Pruning?

Tree Pruning is a technique to reduce the size of a decision tree by removing branches that have little predictive power or that are causing overfitting. A fully-grown decision tree tends to memorize training data (overfitting) — pruning corrects this by simplifying the tree to improve generalization on unseen data.

Types of Pruning

Pre-Pruning (Early Stopping)

Stop growing the tree early. Conditions to stop:

  • Node has fewer than min_samples_split
  • Maximum depth reached
  • Information gain below threshold
Post-Pruning (Bottom-Up)

Grow the full tree, then trim. Methods:

  • Reduced Error Pruning: Remove subtree if accuracy doesn't drop on validation set
  • Cost Complexity Pruning: Add penalty α for tree size
Before Pruning (Overfitted) vs After Pruning (Generalized)
📝 Remember: Pruning = fix for overfitting in decision trees. Pre-pruning = stop early. Post-pruning = cut back after full growth.

What is Hierarchical Clustering?

Hierarchical clustering creates a tree of clusters called a dendrogram. It does not require specifying K in advance. By cutting the dendrogram at different heights, you can obtain any number of clusters — the user chooses the cut level.

Two Approaches

Agglomerative (Bottom-Up) ← Most Common
  1. Start: each point is its own cluster (N clusters)
  2. Merge the two closest clusters into one
  3. Repeat until only one cluster remains
Divisive (Top-Down)
  1. Start: all points in one single cluster
  2. Split into two sub-clusters
  3. Repeat until each point is its own cluster

Linkage Criteria

Linkage MethodDistance Between ClustersCharacteristic
Single LinkageMin distance of any two points across clustersChain-like clusters
Complete LinkageMax distance of any two points across clustersCompact, tight clusters
Average LinkageAverage of all pairwise distancesBalanced clusters
Ward's MethodMinimizes increase in within-cluster varianceMost compact clusters (most common)
Dendrogram — Hierarchical Clustering of 6 Points (A–F) — Agglomerative

How to Determine Number of Clusters

Draw a horizontal line across the dendrogram. The number of vertical lines the cut intersects = number of clusters. Cut at the longest vertical gap for the most natural and well-separated clustering.

Q 13Explain Overfitting Problem. / Discuss ways to avoid overfitting in classification.3 M
Sum'23Win'24Sum'25

What is Overfitting?

Overfitting occurs when a machine learning model learns the training data too well — including its noise and random fluctuations — resulting in excellent training accuracy but poor performance on unseen test data. The model memorizes rather than generalizes.

Sign of overfitting: Training accuracy = 99%, Test accuracy = 60% — a large gap between the two.

Causes

  • Model is too complex (too deep decision tree, too many parameters)
  • Training dataset is too small or not representative of the real population
  • Too many irrelevant features included in the model
  • Training for too many iterations without monitoring validation error

Ways to Avoid Overfitting

  1. Pruning (Decision Trees): Remove branches with little predictive power
  2. Cross-Validation (k-fold): Evaluate model on multiple train/test splits to detect overfitting
  3. Regularization (L1/L2): Add a penalty for model complexity in regression models
  4. More Training Data: Larger, more diverse datasets reduce memorization
  5. Feature Selection: Remove irrelevant and redundant features
  6. Ensemble Methods: Random Forest averages many trees, reducing variance
  7. Early Stopping: Stop training when validation error starts increasing
  8. Dropout (Neural Networks): Randomly deactivate neurons during training
