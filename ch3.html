What is Classification?

Classification is a supervised machine learning task that assigns input data into one of several predefined categories based on learned patterns from labeled training data.

Goal: Learn a mapping function f(X) → Y, where Y is a discrete class label.

Examples: Email → Spam or Not Spam; Tumor → Malignant or Benign; Digit image → 0–9.

Q 2 Discuss the ways to avoid overfitting in classification. 3 M
Sum'23

What is Overfitting?

Overfitting occurs when a model learns the training data too well — including its noise and outliers — resulting in poor performance on new, unseen data.

Underfitting vs Good Fit vs Overfitting

Ways to Avoid Overfitting

  1. Cross-Validation: Use k-fold cross-validation to evaluate model performance on multiple train/test splits.
  2. Regularization: Add a penalty term to the loss function (L1/Lasso, L2/Ridge) to reduce complexity.
  3. Pruning (Decision Trees): Remove branches that have little predictive power to simplify the tree.
  4. More Training Data: More data reduces the chance of memorizing noise.
  5. Reduce Model Complexity: Use fewer features or simpler algorithms.
  6. Early Stopping: Stop training when validation error starts increasing.
  7. Dropout (Neural Networks): Randomly deactivate neurons during training.
  8. Ensemble Methods: Bagging (Random Forest), Boosting average predictions to reduce variance.
📌 Key formula for L2 regularization: Loss = Original Loss + λ·Σ(βᵢ²) where λ controls penalty strength.

Overgeneralization (Underfitting)

Overgeneralization occurs when a model is too simple to capture the underlying patterns. It generalizes too broadly and fails to learn meaningful distinctions, resulting in poor accuracy on both training and test data.

Overfitting (Too Specific)

Model memorizes training data including noise. High train accuracy, low test accuracy.

Overgeneralization (Too General)

Model is too simple, misses real patterns. Low accuracy on both train and test.

Causes of Overgeneralization

  • Too simple model: Using a linear model for non-linear data
  • Insufficient features: Key predictors are missing from the training data
  • Too much regularization: Overly penalizing the model restricts learning
  • Very small training dataset: Not enough data for meaningful patterns
  • Choosing a large k in KNN: Very large k makes boundaries too smooth

How to Fix Overgeneralization

  • Use a more complex/flexible model
  • Add more relevant features (feature engineering)
  • Reduce regularization penalty (lower λ)
  • Collect more and better quality training data
  • Tune hyperparameters (e.g., reduce k in KNN)
Bias-Variance Tradeoff: Overgeneralization = HIGH BIAS. Overfitting = HIGH VARIANCE. The ideal model balances both.
📌 Exam tip: Contrast overfitting and overgeneralization clearly. Mention bias-variance tradeoff for full marks.

Q 5 Discuss overfitting problem. 3 M
Win'24Sum'25

Overfitting Problem

Overfitting is a modeling error where a model learns the training data too precisely — including noise — and fails to generalize to new unseen data. The model performs excellently on training data but poorly on test/validation data.

Causes of Overfitting

  • Model too complex: Too many parameters relative to training samples
  • Insufficient training data: Small dataset forces memorization
  • Training for too long: Iterative models over-train on the same data
  • Noisy features: Irrelevant or redundant features confuse the model
  • No regularization: Without penalty, models grow unchecked in complexity

How to Detect Overfitting

Training vs Validation Accuracy

Training accuracy: 99% | Validation accuracy: 65% → Clear sign of overfitting.

Learning Curves

Training error decreases but validation error stays high or increases.

Solutions

  • Use more training data
  • Apply regularization (L1 / L2)
  • Use simpler models
  • Apply cross-validation
  • Feature selection / dimensionality reduction
  • Early stopping & dropout (neural networks)
High Training Accuracy + Low Test Accuracy = Overfitting
Low Training Accuracy + Low Test Accuracy = Underfitting (Overgeneralization)

Nearest Neighbor Analysis

Nearest Neighbor Analysis is a technique used in classification and regression that makes predictions based on the similarity of data points in the feature space. The core idea: similar inputs should produce similar outputs.

Classification

The majority class among the k nearest neighbors is assigned to the new point (majority voting).

Regression

The average value of the k nearest neighbors is used as the predicted output.

Distance Measures Used

Euclidean: d = √[Σ(xᵢ - yᵢ)²] Manhattan: d = Σ|xᵢ - yᵢ| Minkowski: d = [Σ|xᵢ - yᵢ|ᵖ]^(1/p)

Applications

  • Recommendation systems (find users similar to you)
  • Image recognition (find similar images)
  • Anomaly detection (far neighbors = outliers)
  • Medical diagnosis (find similar patient cases)
📌 Nearest neighbor analysis is the foundation of KNN. It is non-parametric and instance-based (lazy learning).
Q 7 Discuss working of classification in brief. 3 M
Sum'25

Classification — Working Overview

Classification is a two-phase process: Training Phase and Testing Phase.

Phase 1 — Training Phase

  1. Collect and preprocess labeled training data (handle missing values, normalization, encoding)
  2. Choose a classification algorithm (KNN, Decision Tree, Naive Bayes, SVM, etc.)
  3. Train the model by learning patterns and boundaries from labeled data
  4. Evaluate model on validation data; tune hyperparameters

Phase 2 — Testing / Prediction Phase

  1. Input a new unlabeled data instance
  2. Apply the learned model to extract features
  3. Model predicts the class label based on learned patterns
  4. Evaluate performance: Accuracy, Precision, Recall, F1-Score

Evaluation Metrics

Accuracy = (TP + TN) / (TP + TN + FP + FN) Precision = TP / (TP + FP) Recall = TP / (TP + FN) F1-Score = 2 × (Precision × Recall) / (Precision + Recall)

Average Nearest Neighbor (ANN) Algorithm

ANN extends nearest neighbor by using multiple neighbors and averaging their values instead of taking just one nearest neighbor. It reduces the effect of noise from any single neighbor.

Working Steps

  1. Select the number of neighbors k
  2. For a new query point Q, calculate the distance to all training points
  3. Sort all training points by distance (ascending)
  4. Select the k nearest neighbors
  5. For regression: Calculate the average of the k neighbors' output values
  6. For classification: Take the majority vote among k neighbors

Example — Regression using ANN (k=3)

Predict the price of a house with area = 1600 sq ft.

Average Price = (58 + 52 + 62) / 3 = 172 / 3 = 57.33 ₹ Lakh
Predicted price for a 1600 sq ft house = ₹57.33 Lakh

Basic Concept

Based on the assumption: "Objects that are close to each other in the feature space are likely to belong to the same class or have similar output values."

Feature Space

The n-dimensional space formed by all features. Each data point is a position in this space.

Distance Metric

A function that measures "closeness" between two points.

Query Point

The new, unknown data point for which we want to predict the label or value.

k (Neighbors)

Number of nearest neighbors to consider. Small k = sensitive; Large k = smooth but may underfit.

Common Distance Measures

Euclidean Distance: d(p,q) = √[(x₁-x₂)² + (y₁-y₂)²] Manhattan Distance: d(p,q) = |x₁-x₂| + |y₁-y₂| Cosine Similarity: cos(θ) = (A·B) / (|A|·|B|)

Usage / Applications

  • Classification: Assign class based on majority vote of k nearest neighbors
  • Regression: Predict value as average of k nearest neighbors
  • Recommendation Systems: Netflix, Amazon suggest items based on similar users
  • Image Recognition: Match query image to nearest stored images
  • Anomaly Detection: Points far from all neighbors are flagged as anomalies

Q 10 Describe distance measures / nearest neighbor concepts in data science. 4 M
Win'23

Distance Measures in Data Science

Distance measures quantify how "close" or "far apart" two data points are in the feature space. They are fundamental to KNN, clustering, and many other ML algorithms.

1. Euclidean Distance

The straight-line distance between two points in n-dimensional space.

d(p,q) = √[(p₁-q₁)² + (p₂-q₂)² + ... + (pₙ-qₙ)²] = √[Σ(pᵢ - qᵢ)²]

2. Manhattan Distance (City Block)

Sum of absolute differences along each dimension. Like navigating a city grid.

d(p,q) = |p₁-q₁| + |p₂-q₂| + ... + |pₙ-qₙ| = Σ|pᵢ - qᵢ|

3. Minkowski Distance

d(p,q) = [Σ|pᵢ - qᵢ|^r]^(1/r) — When r=1: Manhattan | When r=2: Euclidean

4. Cosine Similarity

cos(θ) = (A · B) / (||A|| × ||B||) — Ranges from 0 (orthogonal) to 1 (identical)

5. Hamming Distance

Number of positions at which two strings differ. Used for categorical/binary data.

d("karolin","kathrin") = 3 (3 positions differ)
Euclidean vs Manhattan Distance
📌 Always normalize/standardize features before applying distance measures — otherwise features with large scales dominate.

Q 12 Explain working of average nearest neighbor algorithm. (Sum'25 variant) 4 M
Sum'25

Average Nearest Neighbor — Detailed Working

ANN makes predictions by computing the mean (average) of k nearest neighbors' outputs rather than using just one nearest neighbor.

  1. Input: Training dataset D, query point Q, and k value
  2. Distance Calculation: Compute distance between Q and every training point
  3. Sort: Arrange all training points in ascending order of distance from Q
  4. Select k Neighbors: Take the top k closest data points
  5. Average (Regression): Prediction = (sum of k neighbors' Y values) / k
  6. Output: Return the averaged prediction as the result
ANN Prediction = (y₁ + y₂ + y₃ + ... + yₖ) / k Weighted ANN = Σ(wᵢ × yᵢ) / Σwᵢ where wᵢ = 1/dᵢ
k too small (e.g., k=1)

Highly sensitive to noise. Overfitting risk.

k too large (e.g., k=N)

Prediction becomes the global average. Underfitting.

Rule of thumb: k = √n where n = number of training samples. Use cross-validation to find optimal k.

📌 Weighted ANN outperforms simple ANN when training data is dense, as it properly accounts for proximity.

Introduction to K-Nearest Neighbor (KNN)

KNN is a simple, non-parametric, lazy learning algorithm used for both classification and regression. It classifies a new data point based on the majority class of its k nearest neighbors in the training dataset.

Type

Supervised Learning, Instance-based (Lazy Learner)

Lazy Learner Meaning

No explicit model is trained. All computation happens at prediction time.

Non-Parametric

Makes no assumptions about the underlying data distribution.

Key Parameter

k = number of neighbors. Critical hyperparameter affecting accuracy.

KNN Algorithm — Step by Step

  1. Choose the value of k (e.g., k = 3)
  2. For each new data point, compute the distance to all training data points using Euclidean distance
  3. Sort all training points by distance in ascending order
  4. Select the k nearest training points (smallest distances)
  5. For Classification: Assign the class that appears most frequently (majority voting)
  6. For Regression: Predict the mean of the k neighbors' output values

Distance Formula Used

Euclidean Distance: d(p, q) = √[(p₁−q₁)² + (p₂−q₂)² + ... + (pₙ−qₙ)²]

Supervised Learning — Deep Dive

Each example is paired with the correct output (label). It learns a mapping function from inputs to outputs by minimizing prediction error.

Q 15 Describe Normal, Binomial and Categorical distributions. / Differentiate MultinomialNB, BernoulliNB and GaussianNB. 3 M
Sum'24

1. Normal (Gaussian) Distribution

A continuous probability distribution that is symmetric and bell-shaped about the mean μ. Completely described by mean (μ) and standard deviation (σ).

f(x) = (1/σ√2π) · e^[−(x−μ)²/2σ²] Properties: • Mean = Median = Mode (symmetric) • 68% data within 1σ, 95% within 2σ, 99.7% within 3σ • Used in GaussianNB classifier

2. Binomial Distribution

A discrete distribution modeling the number of successes in n independent Bernoulli trials.

P(X = k) = C(n,k) · pᵏ · (1−p)^(n−k) Example: P(3 heads in 5 coin flips) = C(5,3)·(0.5)³·(0.5)² Used in BernoulliNB (binary features: present/absent)

3. Categorical / Multinomial Distribution

Extends binomial to k greater than 2 outcomes. Models the counts of occurrences of multiple categories.

Used in MultinomialNB (word count features in text classification)
