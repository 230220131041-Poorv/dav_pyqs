Six Steps of Data Visualization Process

  1. Acquire the Data: Collect raw data from databases, APIs, spreadsheets, surveys, or other sources. Ensure data quality and completeness.
  2. Parse / Prepare the Data: Clean, format, and structure the data. Handle missing values, remove duplicates, normalize if needed.
  3. Filter the Data: Select only the relevant subset of data needed for the specific visualization. Remove noise and irrelevant records.
  4. Mine the Data (Analyze): Apply statistical methods, identify patterns, correlations, trends, and outliers in the filtered dataset.
  5. Represent the Data: Choose the appropriate chart or graphic type (bar, line, scatter, map, etc.) based on the data type and message.
  6. Refine and Interact: Improve aesthetics, add interactivity (tooltips, filters, zoom), and present the final visualization to the target audience.

Time Series Analysis in Data Visualization

A time series is a sequence of data points indexed in time order. In the context of data visualization, time series data is one of the most common types visualized — it shows how a variable changes over time.

Visualization Techniques for Time Series

  • Line Chart: Most common — connects data points along a continuous time axis to show trends
  • Area Chart: Line chart with the area below filled — emphasizes volume/magnitude over time
  • Candlestick Chart: Used in finance — shows open, close, high, low values per time period
  • Heatmap Calendar: Shows daily values across months using color intensity (e.g., GitHub activity)
  • Horizon Chart: Compact multi-layer view for large time series data

Key Components Visualized

Trend

Long-term upward or downward direction shown by a smooth line.

Seasonality

Repeating periodic patterns highlighted by overlaying multiple years.

Anomalies

Outlier spikes highlighted by annotations or reference lines.

Forecast

Predicted future values shown as a dotted line with confidence interval.

What is a Data Graphic?

A data graphic is any visual representation that encodes data values into visual properties — such as position, length, area, color, or shape — to communicate information efficiently. Data graphics translate abstract numbers into meaningful visual patterns that the human brain can interpret rapidly.

Components of a data graphic:

  • Visual marks: Points, lines, bars, areas — the basic geometric elements
  • Visual channels: Position, size, color, shape, angle — properties used to encode data
  • Axes and labels: Reference system that decodes the encoding
  • Title and annotations: Context that guides interpretation

Comparative Graphics

Comparative graphics are data visualizations specifically designed to compare values across categories, groups, or time periods. They answer questions like: "Which is bigger?", "How do these differ?", "How has this changed?"

Types of Comparative Graphics

Bar Chart

Compare values across discrete categories. Horizontal bars work better for long category names.

Grouped Bar Chart

Compare multiple series side-by-side within each category (e.g., sales by product per quarter).

Stacked Bar Chart

Show both total and part-to-whole comparison. Bars are divided into segments.

Bullet Chart

Compare an actual value against a target. Used in dashboards for KPI tracking.

Slope Graph

Show change between two time points for multiple items. Lines connect before and after values.

Dot Plot

Compare values across many categories more cleanly than a bar chart. Reduces visual clutter.

Comparative Graphics — Bar Chart vs Grouped Bar Chart
📝 Best practice: Use horizontal bars when category names are long. Always start bar charts at zero — truncating the axis is misleading.

Why Design Style Matters

In data visualization, design style is not just about aesthetics — it directly affects how effectively information is communicated. The wrong style can mislead, confuse, or disengage the audience, even if the data is accurate.

Factors Influencing Style Choice

Audience

Experts need precise, dense charts. General public needs simple, annotated visuals. Executives prefer high-level dashboards.

Purpose / Goal

Exploring data? Use interactive style. Presenting findings? Use narrative style. Monitoring KPIs? Use dashboard style.

Data Type

Time data → Line chart. Categories → Bar chart. Relationships → Scatter. Geography → Map. Parts of whole → Pie.

Medium / Channel

Print: static, no interactivity. Web: can be interactive. Mobile: compact layout needed.

Importance — Why Getting It Right Matters

  1. Clarity of message: The right chart type makes the key insight immediately obvious
  2. Audience engagement: Appropriate style keeps the audience interested and avoids cognitive overload
  3. Credibility: A well-designed visualization signals professionalism and trustworthiness
  4. Avoiding misinterpretation: Wrong style choices (e.g., 3D pie charts, truncated axes) distort perception of data
  5. Accessibility: Color-blind-safe palettes and proper contrast make visualizations usable by all

Data-to-Ink Ratio (Tufte's Principle)

Edward Tufte's key rule: maximize the proportion of ink (pixels) used to display data, not decoration. Remove chart junk — unnecessary gridlines, borders, legends, shadows — that add noise without adding information.

Data-to-Ink Ratio = Data Ink / Total Ink Used in Graphic → Maximize this ratio for effective design

What is D3.js?

D3.js (Data-Driven Documents) is an open-source JavaScript library for creating dynamic, interactive data visualizations in web browsers. It uses HTML, SVG, and CSS to bind data to the DOM and apply transformations to create charts and graphics.

D3.js = Data-Driven Documents Created by: Mike Bostock (2011) Language: JavaScript Renders: SVG, HTML, Canvas

Purpose of D3.js

  1. Data Binding: D3 binds data directly to DOM elements, allowing any change in data to automatically update the visualization.
  2. Custom Visualizations: Unlike chart libraries (Chart.js), D3 gives full control to create any type of visualization — not limited to standard charts.
  3. Interactivity: Supports user interactions like hover effects, tooltips, zoom, pan, and click events using JavaScript event listeners.
  4. Animations and Transitions: Smooth animated transitions when data changes, making visualizations more engaging.
  5. Scalable Vector Graphics (SVG): Renders resolution-independent graphics that scale perfectly on any screen size.
  6. Wide Chart Library: Supports bar charts, line charts, scatter plots, geographic maps, tree diagrams, chord diagrams, force-directed graphs, and more.

Why D3.js over other libraries?

PerformanceHigh (SVG + Canvas)Good for small datasets

Uses a matrix of colored cells where color intensity represents values. Best for: showing patterns in a 2D matrix (e.g., correlation matrix, time vs. day activity).

8. Area Chart

Like a line chart but with the area below the line filled. Useful for showing cumulative totals or volume over time.

Standard Charts in Data Visualization

Standard charts are the universally recognized, tried-and-tested chart types that most audiences already know how to read. They form the core toolkit of any data visualization practitioner.

Bar / Column Chart

Vertical (column) or horizontal bars encoding values by length. Best for comparing discrete categories. Rule: always start Y-axis at 0.

Line Chart

Connects ordered data points with a line. Ideal for continuous data and time series. Multiple lines can compare several series.

Pie / Donut Chart

Circular slices proportional to values. Use only when showing parts of a whole with few categories (≤5). Donut adds center label space.

Scatter Plot

Two-dimensional plot of numerical data. Each point = one observation. Reveals correlation, clustering, and outliers between variables.

Histogram

Bar chart for continuous data grouped into intervals (bins). Shows the distribution shape — normal, skewed, bimodal, etc.

Area Chart

Filled line chart emphasizing magnitude over time. Stacked area charts show composition changes over time.

📝 Standard charts are preferred for general audiences because they require no learning. Use non-standard (chord, radial, Sankey) only for expert audiences or when no standard chart fits.
Q 13 What are the significance of SVG and DOM in D3.js? 4 M
Sum'24 OR

DOM (Document Object Model) in D3.js

The DOM is a tree-like representation of an HTML page where every element (div, p, svg, rect, circle) is a node. D3 manipulates the DOM directly to create, update, and remove visual elements based on data.

  • D3 uses d3.select() and d3.selectAll() to select DOM elements
  • Once selected, D3 can set attributes, styles, text content, and event handlers on those elements
  • The DOM update cycle (Enter → Update → Exit) is the core mechanism of D3's data-driven approach
DOM Manipulation Example: d3.select("body") // Select the body element
.append("p") // Add a paragraph to DOM
.text("Hello D3"); // Set its text content

SVG (Scalable Vector Graphics) in D3.js

SVG is an XML-based vector image format for 2D graphics. D3 primarily renders visualizations as SVG elements because:

  • Scalability: SVG graphics look sharp at any screen resolution (unlike Canvas/bitmap)
  • DOM integration: SVG elements are part of the DOM — each rect, circle, line is a selectable node that D3 can bind data to
  • CSS styling: SVG elements can be styled with CSS (fill, stroke, opacity)
  • Event handling: Each SVG element can have click, hover, mousemove listeners attached
SVG Elements used in D3: <svg> — container
<rect> — bars in bar chart
<circle> — points in scatter plot
<line> — axes, connectors
<path> — lines, areas, arcs
<text> — labels, titles

How SVG + DOM + D3 Work Together

D3 creates SVG elements inside the DOM, binds data to them, and updates their visual properties (x, y, width, fill) based on data values — creating a fully data-driven visualization.

📝 Key point: In D3, every visual element (bar, circle, line) is an SVG element that lives in the DOM — making it easy to select, update, animate, and add interactivity.

D3.js is used to create web-based data visualizations by binding data to DOM elements and applying transformations. It handles the full pipeline: data loading → binding → rendering → interaction → animation.

Typical D3.js Workflow

  • Steep learning curve — complex API
  • Verbose code for simple charts
  • No built-in chart templates

  • Debugging SVG/DOM issues is harder

Comparative Graphics

Comparative graphics are data visualizations specifically designed to help the viewer compare values, quantities, or distributions across different groups, categories, or time periods. They are the most common class of charts used in business and reporting.

Key Comparative Chart Types

1. Bar Chart (most common)

Encodes values as bar lengths. Simple, familiar, effective for comparing 2–20 categories.

2. Grouped Bar Chart

Places bars from multiple series side-by-side for the same category. Good for comparing 2–3 series across categories.

3. Stacked Bar Chart

Bars are divided into segments showing sub-components. Compares totals and composition simultaneously.

4. Slope Graph

Shows how values changed between two time points for multiple entities. Easy to see who went up/down.

5. Lollipop Chart

A cleaner version of bar chart with a dot at the end of a thin line. Reduces visual clutter while comparing values.

6. Small Multiples (Trellis Chart)

Shows the same chart type repeated for different subgroups, allowing easy comparison of patterns across groups.

Design Guidelines for Comparative Graphics

  • Always start bar charts from zero to avoid visual distortion
  • Use consistent colors for the same category across all charts
  • Sort bars by value (not alphabetically) to make comparisons easier
  • Limit colors — use a single color family with varying shades, or highlight only the key bar

The Big Three of Data Visualization

The "Big Three" is a framework that classifies all data visualizations into three fundamental types based on their purpose, audience, and design philosophy. Understanding which category a visualization belongs to is essential for choosing the right approach.

The Big Three — Purpose Spectrum

1. Data Storytelling

Definition: Data storytelling is the practice of building a narrative around data insights to communicate a clear message to a specific audience. It combines data, narrative, and visuals to explain what happened, why it matters, and what should be done.

Key Characteristics:

  • Has a clear beginning, middle, and end (narrative arc)
  • Designed for a specific audience with a specific goal
  • Uses annotations, callouts, and highlights to guide attention
  • The visualization serves the story — not the other way around

Tools & Charts Used: Annotated line charts, slope graphs, small multiples, infographics, scrollytelling web pages.

Example: A news article that uses animated charts to tell the story of how COVID-19 spread across continents, with annotations at key events.

2. Data Showcasing

Definition: Data showcasing presents data clearly, accurately, and completely so that the audience can explore it, draw their own conclusions, and make decisions. The focus is on the data itself, not on a predetermined story.

Key Characteristics:

  • Designed for analysts, experts, or decision-makers
  • Interactive — allows filtering, drilling down, zooming
  • Multiple perspectives shown simultaneously (dashboards)
  • Accuracy and completeness are paramount

Tools & Charts Used: Interactive dashboards (Tableau, Power BI), scatter plots, heatmaps, box plots, pivot tables.

Example: A sales dashboard showing revenue by region, product, and time period with interactive filters for the sales team.

3. Data Art

Definition: Data art uses data as raw creative material to produce visually stunning or emotionally evocative works where aesthetic impact is the primary goal. Strict accuracy or information transfer may be secondary to visual beauty.

Key Characteristics:

  • Prioritizes aesthetic experience over information clarity
  • Pushes creative and technical boundaries
  • Often displayed in galleries, exhibitions, or public installations
  • May use physical media — LED displays, 3D printing, projections

Tools & Charts Used: Network graphs, chord diagrams, generative art algorithms, Processing/p5.js, custom D3 visualizations.

Example: A visualization of all Wikipedia edits as a flowing river of colored streams, each color representing a different contributor country — displayed as a gallery installation.

Introduction

Before creating any data visualization, the designer must go through three critical pre-design steps: Brainstorm, Define the Purpose, and Choose a Functional Design. These steps ensure the final visualization serves its intended audience effectively.

Step 1 — Brainstorm

Brainstorming is the open, creative exploration phase where all ideas about what to visualize and how are considered without judgment.

Activities during brainstorming:

  • Identify all available data sources and what data you have
  • List all potential insights, questions, or stories the data could tell
  • Sketch rough ideas on paper — no need for perfection at this stage
  • Consider different chart types, layouts, and perspectives
  • Think about the context: What is happening? Why? So what?
  • Brainstorm with the team — different viewpoints lead to better ideas

Output: A list of possible visualization concepts and rough sketches.

Step 2 — Define the Purpose

Defining purpose means answering the fundamental question: "Why are we creating this visualization and for whom?" Without a clear purpose, even beautiful visualizations fail to communicate.

Key questions to answer:

  • What is the message? — What single insight or story should the viewer take away?
  • Who is the audience? — Executives? Scientists? The general public? Children?
  • What is the context? — Where will it be shown? Print, web, presentation, mobile?
  • What action should it trigger? — Awareness? Decision? Behavior change?
  • What type of visualization is needed? — Storytelling, Showcasing, or Data Art?

Output: A clear purpose statement, e.g.: "Show the sales team which products underperformed last quarter so they can plan corrective actions."

Step 3 — Choose a Functional Design for Target Audience

Once the purpose is clear, the designer selects the visual form, chart type, color palette, and level of interactivity appropriate for the target audience.

Audience-Based Design Choices

Target AudienceDesign ApproachChart Types
General PublicSimple, annotated, colorful, minimal jargonBar, Pie, Infographic
Business ExecutivesHigh-level summary, KPIs, dashboard styleGauge, Scorecard, Summary table
Data AnalystsDetailed, interactive, allow explorationScatter, Heatmap, Box plot
Scientists / ResearchersPrecise, minimal decoration, error bars shownError bars, confidence intervals, histograms

Functional design principles for any audience:

  • Use pre-attentive attributes (color, size, position) to draw attention to key data
  • Eliminate chart junk — every element must earn its place
  • Use color purposefully — not decoratively
  • Ensure accessibility (color-blind safe palettes, alt-text, sufficient contrast)
  • Test with a real member of the target audience before finalizing

Q 18 Explain how D3.js can be used to create interactive data visualizations. Provide a simple example. 7 M

D3.js for Interactive Visualizations

D3.js enables interactivity in data visualizations by attaching JavaScript event listeners to SVG elements in the DOM. Common interactions include: tooltips on hover, highlighting on click, zoom and pan, animated transitions, and data filtering via controls.

Key Interactivity Mechanisms in D3

Event Listeners

.on("mouseover", fn), .on("click", fn), .on("mouseout", fn) — attach browser events to any SVG element.

Transitions

.transition().duration(500) — smoothly animate changes in position, size, color when data updates.

Tooltips

A hidden HTML div shown near the cursor on hover, displaying data values for the element under the mouse.

Zoom / Pan

d3.zoom() — allows users to zoom in and pan around large charts (maps, scatter plots with many points).

Simple Example — Interactive Bar Chart with Tooltip

// 1. Data const data = [
{ city: "Ahmedabad", temp: 38 },
{ city: "Mumbai", temp: 32 },
{ city: "Delhi", temp: 41 },
{ city: "Chennai", temp: 35 }
];

// 2. SVG Setup
const svg = d3.select("#chart")
.append("svg")
.attr("width", 500)
.attr("height", 300);

// 3. Scales
const xScale = d3.scaleBand()
.domain(data.map(d => d.city))
.range([0, 500]).padding(0.3);

const yScale = d3.scaleLinear()
.domain([0, 50]).range([250, 0]);

// 4. Tooltip (hidden div)
const tooltip = d3.select("body").append("div")
.style("position","absolute")
.style("background","#333")
.style("color","#fff")
.style("padding","6px 10px")
.style("border-radius","4px")
.style("display","none");

// 5. Bars with Interactivity
svg.selectAll("rect")
.data(data)
.enter()
.append("rect")
.attr("x", d => xScale(d.city))
.attr("y", d => yScale(d.temp))
.attr("width", xScale.bandwidth())
.attr("height", d => 250 - yScale(d.temp))
.attr("fill", "#3a6b3a")
.on("mouseover", function(event, d) {
d3.select(this).attr("fill", "#b04a1a"); // Highlight
tooltip.style("display","block")
.html(d.city + ": " + d.temp + "°C")
.style("left", (event.pageX+10)+"px")
.style("top", (event.pageY-20)+"px");
})
.on("mouseout", function() {
d3.select(this).attr("fill", "#3a6b3a"); // Restore
tooltip.style("display","none");
});

// 6. Axes
svg.append("g").attr("transform","translate(0,250)")
.call(d3.axisBottom(xScale));
svg.append("g").call(d3.axisLeft(yScale));

What the Example Does

  • Creates a bar chart of city temperatures using D3 scales
  • On mouseover: bar turns orange and a tooltip shows the city name and temperature
  • On mouseout: bar returns to green and tooltip hides
  • Axes are drawn using D3's built-in axis generator

Design Principles for Data Visualization

Effective data visualization is governed by a set of well-established design principles. Following these principles ensures that visualizations are clear, accurate, and meaningful to the target audience.

1. Principle of Clarity (Simplicity)

Every element of the visualization must serve a purpose. Remove anything that does not directly help the viewer understand the data. Edward Tufte's concept of "chart junk" — unnecessary gridlines, borders, 3D effects, decorative icons — must be eliminated.

  • Use a clear, descriptive title
  • Label axes directly; minimize legends when possible
  • Use white space generously to separate elements

2. Principle of Accuracy

Visualizations must faithfully represent the data. Distortions — intentional or accidental — mislead viewers.

  • Bar charts must start at zero (truncated axes exaggerate differences)
  • Pie charts must sum to 100%
  • Area must be proportional to values (avoid 3D charts that distort area)
  • Scale must be consistent across all comparable charts

3. Principle of Efficiency (Data-to-Ink Ratio)

Tufte's data-to-ink ratio: Maximize the proportion of "ink" (pixels) used to represent data, not decoration. A well-designed chart shows maximum information with minimum visual complexity.

4. Principle of Appropriate Chart Choice

Choose the chart type based on what the data is and what comparison you want to show.

Data RelationshipBest Chart
Compare categoriesBar chart
Show trend over timeLine chart
Part-to-wholePie / Donut chart (≤5 slices)
CorrelationScatter plot
DistributionHistogram / Box plot
Geographic dataChoropleth map

5. Principle of Color Use

Color is one of the most powerful visual channels but is often misused.

  • Use color to encode data: not for decoration
  • Sequential palette: for ordered data (light to dark)
  • Diverging palette: for data with a meaningful midpoint (e.g., profit/loss)
  • Categorical palette: for nominal groups (visually distinct hues)
  • Always check color-blind accessibility (avoid red-green alone)
  • Limit to 5–7 colors maximum in one chart

6. Principle of Pre-attentive Attributes

Pre-attentive attributes are visual properties that the brain processes automatically, before conscious thought — in under 500ms. Use them to direct the viewer's attention instantly.

Pre-attentive attributes:

Color hue, Color intensity, Size, Length, Orientation, Shape, Motion, Position

Application:

Highlight one bar in orange while all others are grey → eye goes there instantly. Use size to show magnitude without labels.

7. Principle of Audience-First Design

Every design decision — chart type, color, complexity, level of detail — must be made based on who will view the visualization and in what context. A chart perfect for a data scientist may completely fail with a general audience.

8. Gestalt Principles of Visual Perception

  • Proximity: Elements close together are perceived as related
  • Similarity: Elements with same color/shape are perceived as a group
  • Enclosure: Elements surrounded by a border are seen as one group
  • Continuity: Eyes follow continuous lines or curves naturally

What is D3.js?

D3.js (Data-Driven Documents) is a JavaScript library for creating dynamic, interactive, and data-driven visualizations in web browsers. It manipulates the DOM by binding datasets to HTML/SVG elements and applying transformations based on data values.

Created by Mike Bostock in 2011. Uses web standards: HTML, SVG, CSS, and JavaScript.

Core D3 Methods — Selection and Data Binding

1. d3.select() — Single Element Selection

d3.select(selector) selects the first matching DOM element and returns a D3 selection object. Used to select a container, a specific element, or to modify one element.

// Select the first <p> element on the page
const para = d3.select("p");

// Select by ID
const chart = d3.select("#myChart");

// Select by class
const title = d3.select(".chart-title");

// Chain methods to modify it
d3.select("body")
.style("background-color", "#f5f5f5")
.append("h1")
.text("My D3 Chart");

Key point: select() returns a D3 selection wrapping a single element. All subsequent chained methods apply to that element.

2. d3.selectAll() — Multiple Element Selection

d3.selectAll(selector) selects all matching DOM elements and returns a D3 selection object containing all of them. Used to manipulate or bind data to multiple elements at once.

// Select ALL <circle> elements in the SVG
const circles = d3.selectAll("circle");

// Select all elements with class "bar"
const bars = d3.selectAll(".bar");

// Apply attributes to ALL selected elements
d3.selectAll("rect")
.attr("fill", "#3a6b3a")
.attr("rx", 4)
.style("opacity", 0.8);

// Count: selection.size()
console.log(bars.size()); // number of matched elements

Key point: selectAll() is used before .data() to prepare for data binding — it creates the set of DOM elements that will be matched with data points.

3. .data() — Data Binding

.data(dataset) is the most important D3 method. It binds an array of data values to a selection of DOM elements, creating a one-to-one mapping between data points and visual elements.

const dataset = [30, 50, 80, 45, 60];

// Bind data to rect elements
const bars = d3.select("svg")
.selectAll("rect") // Select all rects (may not exist yet)
.data(dataset); // Bind data array to selection

// .enter() handles new elements (data without DOM elements)
bars.enter()
.append("rect") // Create a <rect> for each data point
.attr("x", (d, i) => i * 60) // d = data value, i = index
.attr("y", d => 200 - d) // Use data value for height
.attr("width", 50)
.attr("height", d => d) // Bar height = data value
.attr("fill", "#3a6b3a");

The Enter-Update-Exit Pattern

When .data() is called, D3 computes three sets:

SetMeaningHandled by
EnterData points with no corresponding DOM element yet.enter().append()
UpdateData points that already have a DOM elementDirect attribute calls on selection
ExitDOM elements with no corresponding data point.exit().remove()
// Full Enter-Update-Exit pattern
const rects = svg.selectAll("rect").data(newData);

rects.enter().append("rect") // NEW elements
.attr("fill","#3a6b3a");

rects.attr("width", d => d); // UPDATE existing elements

rects.exit().remove(); // REMOVE elements with no data
📝 Key: select() = one element, selectAll() = many elements, data() = bind array to selection. The callback (d, i) in attribute functions gives you the data value (d) and index (i) for each element.

Target Audience in Data Visualization

The target audience is the single most important factor in designing a data visualization. A chart that is perfect for a data scientist can completely fail with a non-technical executive, and vice versa. The audience determines: chart complexity, level of interactivity, amount of annotation, use of jargon, and design style.

Types of Target Audience

1. General Public / Lay Audience

  • Background: No technical expertise, limited time, varied literacy levels
  • Needs: Immediate clarity, simple narrative, minimal jargon, emotional engagement
  • Suitable Graphics: Simple bar charts, pie charts, pictographs, annotated infographics, icon arrays
  • Design approach: Bold headlines, large fonts, strong colors, one key insight only, annotations that explain what the viewer is seeing
  • Example context: Newspaper article, social media post, government health campaign

2. Business Executives / Decision Makers

  • Background: Domain experts, time-pressed, high-level view preferred
  • Needs: KPIs, summary metrics, trend direction, comparison to targets
  • Suitable Graphics: Dashboards with gauge charts, scorecards, sparklines, bullet charts, traffic-light indicators
  • Design approach: Compact, clean, highlights exceptions only, clear comparison to benchmark
  • Example context: Executive dashboard, board presentation, weekly business review

3. Data Analysts / Technical Experts

  • Background: Statistical literacy, comfortable with complexity, want to explore data
  • Needs: Full detail, interactivity, multiple views, raw data access
  • Suitable Graphics: Scatter plots, box plots, heatmaps, histograms, parallel coordinates, interactive filters
  • Design approach: Dense information, interactive, allow drill-down, show distributions and outliers
  • Example context: Analytics platform, Jupyter notebook, Tableau workbook for data team

4. Scientific / Academic Audience

  • Background: Deep domain expertise, value precision over aesthetics
  • Needs: Statistical rigor, confidence intervals, error bars, reproducibility
  • Suitable Graphics: Histograms with overlaid density curves, forest plots, error bar charts, survival curves, Q-Q plots
  • Design approach: Minimal decoration, precise labeling, include sample size and p-values
  • Example context: Research paper figure, scientific journal, conference poster

5. Children / Students

  • Background: Limited prior knowledge, short attention spans, learn through engagement
  • Needs: Fun, relatable context, simple concepts, visual delight
  • Suitable Graphics: Pictographs (icons instead of bars), animated charts, color-coded simple bar/pie, story-based visualizations
  • Design approach: Bright colors, familiar images, very few data points, gamified if possible
  • Example context: Educational app, classroom textbook, museum exhibit

D3-DOM Selection Methods

D3 provides a powerful set of methods to select, manipulate, and create DOM elements. These are the foundation of all D3 visualizations.

1. d3.select(selector)

Selects the first DOM element matching the CSS selector. Returns a D3 selection object for one element.

d3.select("body") // Select <body> tag
d3.select("#chart") // Select element with id="chart"
d3.select(".title") // Select first element with class="title"
d3.select("svg > rect") // Select first rect inside svg

2. d3.selectAll(selector)

Selects all DOM elements matching the selector. Returns a selection wrapping all matched elements.

d3.selectAll("circle") // Select ALL circle elements
d3.selectAll(".bar") // Select ALL elements with class bar
d3.selectAll("g.axis") // All g elements with class axis

// Apply to all elements at once
d3.selectAll("rect")
.attr("rx", 4) // Round all bar corners

3. selection.append(elementType)

Appends a new child DOM element to each element in the selection. Returns a selection of the newly created elements.

// Add an SVG to the body
const svg = d3.select("body")
.append("svg")
.attr("width", 500)
.attr("height", 300);

// Add a circle inside the SVG
svg.append("circle")
.attr("cx", 100).attr("cy", 100).attr("r", 40)
.attr("fill", "#3a6b3a");

4. selection.attr(name, value)

Gets or sets an HTML/SVG attribute. When value is a function, it receives (d, i) — data value and index — for each element.

selection.attr("width", 50) // Set fixed width
selection.attr("x", (d, i) => i * 60) // Computed from index
selection.attr("height", d => d.value) // Computed from data

5. selection.style(name, value)

Sets a CSS style property on selected elements.

d3.select("svg")
.style("border", "1px solid #ddd")
.style("background", "#f9f9f9");

6. selection.text(value)

Sets the text content of selected elements.

d3.selectAll("text")
.text(d => d.label) // Set text from data value

7. selection.remove()

Removes selected elements from the DOM. Used in the Exit pattern when data is removed.

d3.selectAll(".bar").exit().remove() // Remove bars with no data

8. selection.on(event, handler)

Attaches an event listener to selected elements for interactivity.

d3.selectAll("rect")
.on("mouseover", function(event, d) {
d3.select(this).attr("fill", "#b04a1a");
console.log("Value:", d);
})
.on("mouseout", function() {
d3.select(this).attr("fill", "#3a6b3a");
});

Complete Mini-Example — All Methods Together

const data = [40, 70, 55, 90, 30];

const svg = d3.select("#chart").append("svg")
.attr("width", 300).attr("height", 150);

svg.selectAll("rect") // selectAll
.data(data) // bind data
.enter() // enter (new elements)
.append("rect") // append rect for each
.attr("x", (d, i) => i*55) // attr using index
.attr("y", d => 130 - d) // attr using data value
.attr("width", 45)
.attr("height", d => d)
.attr("fill","#3a6b3a")
.on("mouseover", function() {
d3.select(this).style("opacity", 0.6); // style
})
.on("mouseout", function() {
d3.select(this).style("opacity", 1);
});

svg.selectAll("text")
.data(data).enter()
.append("text")
.attr("x", (d, i) => i*55 + 22)
.attr("y", d => 125 - d)
.text(d => d) // text from data
.attr("text-anchor","middle")
.attr("font-size", 11);
